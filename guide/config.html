翻译功能设计

本页目录

配置说明 ​

hias-cli 使用项目级 .hias/setting.json 管理翻译行为。

创建配置 ​

sh
hias setting

配置文件通常位于当前业务项目：

text
.hias/setting.json

翻译服务商 ​

示例：

json
{
  "translationSetting": {
    "sourceLocale": "",
    "locales": ["zh-CN", "en-US"],
    "provider": "baidu",
    "replaceOriginalFile": false,
    "pruneUnusedKeys": false,
    "keyStrategy": {
      "maxLength": 40,
      "collision": "number",
      "hashLength": 6
    },
    "appId": "your_app_id",
    "secretKey": "your_secret_key"
  }
}

没有密钥时，工具仍可提取中文并生成语言包，只是目标语言可能暂时回退为中文原文。

语言方向 ​

sourceLocale 表示源码当前使用的语言，locales 表示需要生成的语言集合。

兼容旧配置时可以不写 sourceLocale，工具会继续把 locales[0] 当源语言：

json
{
  "translationSetting": {
    "locales": ["zh-CN", "en-US", "ja-JP"]
  }
}

如果源码是英文，希望生成中文和日文：

json
{
  "translationSetting": {
    "sourceLocale": "en-US",
    "locales": ["zh-CN", "en-US", "ja-JP"]
  }
}

此时目标语言会自动变成 zh-CN 和 ja-JP。

常用字段 ​

字段默认值说明
sourceLocale""源码当前语言。留空时使用 locales[0] 兼容旧配置
locales["zh-CN", "en-US"]要生成的语言集合，目标语言会排除源语言
outDir.hias/lang翻译副本和语言包输出目录
replaceOriginalFilefalse是否直接覆盖原文件
i18nCallTemplate$t替换时使用的 i18n 调用模板
extensions[]自定义扫描扩展名；空数组表示使用全部支持类型
capitalizeTranslationsfalse是否将含拉丁字符的翻译结果首字母大写
pruneUnusedKeysfalse写入语言包时是否移除当前命名空间未引用 key
keyStrategy.maxLength40自动 key 最大长度
keyStrategy.collisionnumberkey 冲突策略，支持 number 和 hash
keyStrategy.hashLength6hash 冲突策略的短 hash 长度

Key 策略 ​

默认使用翻译后的英文生成 snake_case key，并在冲突时追加序号：

json
{
  "translationSetting": {
    "keyStrategy": {
      "maxLength": 40,
      "collision": "number",
      "hashLength": 6
    }
  }
}

如果你更希望多次运行时 key 后缀稳定，可以改为 hash：

json
{
  "translationSetting": {
    "keyStrategy": {
      "collision": "hash"
    }
  }
}

未使用 key 清理 ​

默认会保留语言包中已有 key，避免删除人工维护的历史翻译。确认要清理当前命名空间未引用 key 时，再开启：

json
{
  "translationSetting": {
    "pruneUnusedKeys": true
  }
}

翻译缓存 ​

缓存文件位于：

text
.hias/.translation-cache.json

缓存 key 使用语言方向：

json
{
  "zh-CN|en-US": {
    "登录": "Login"
  }
}

推荐通过 hias tci 导入，而不是手写缓存文件。

输出目录 ​

翻译结果通常写入 locale module 目录，例如：

text
src\locale\module\vueAll
.hias\locale\module\jsxAll

实际路径以项目配置为准。

CJS 项目和 VitePress ​

当前 hias-cli 主项目保持 CommonJS 规范，不需要在根 package.json 添加 "type": "module"。

文档站配置使用：

text
docs/.vitepress/config.mts

这样 VitePress 可以使用 ESM 配置，而 CLI 的 require、module.exports、build.config.js 不受影响。
